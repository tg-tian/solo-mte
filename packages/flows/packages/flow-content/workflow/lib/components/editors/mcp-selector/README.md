# MCP Selector Component

## 概述

MCP选择器组件用于选择和管理MCP（Model Context Protocol）工具，提供丰富的分类层级和工具选择体验。当真实的MCP服务接口尚未发布时，组件会自动使用Mock数据来提供完整的功能演示。

## 功能特性

### ✅ 核心功能
- **丰富的分类层级** - 5个一级分类，每个一级分类下有3个二级分类，总共16个分类
- **原始选择界面** - 恢复简洁直观的工具选择界面，支持下拉分类筛选
- **即时工具展示** - 选择分类后立即显示工具列表，支持服务器详情展示
- **灵活筛选机制** - 支持一级和二级分类筛选，快速定位目标工具
- **动态参数生成** - 根据JSON Schema自动生成对应的输入参数

### 操作流程
1. **选择分类** - 在下拉框中选择分类（支持一级和二级分类筛选）
2. **查看服务器** - 分类筛选后显示对应的服务器列表
3. **选择服务器** - 点击服务器查看工具列表
4. **选择工具** - 在工具列表中选择具体工具
5. **自动配置** - 选择工具后自动配置输入参数并关闭弹窗

```typescript
if (apiError) {

  console.warn('真实API不可用，使用Mock数据');
}
```

## Mock数据统计

### 分类数据
- **一级分类**: 5个（数据处理、文件操作、网络通信、数据库操作、AI处理）
- **二级分类**: 11个，每个一级分类下有2-3个二级分类
- **总分类数**: 16个

### 服务器数据
- **服务器总数**: 7个
- **总工具数**: 8个（不同服务器可能有重复工具）
- **覆盖分类**: 所有主要分类都有对应的服务器

### 工具类型分布
- **数据处理类工具**: 4个（CSV解析、JSON格式化、XML解析、文本摘要）
- **文件操作类工具**: 2个（Excel读取、图像尺寸调整）
- **网络通信类工具**: 2个（HTTP请求、API调用）
- **AI处理类工具**: 2个（文本摘要、图像处理）
- **数据库类工具**: 1个（SQL查询执行）

## 注意事项

- Mock数据包含完整的Schema定义，支持string、number、boolean、array、object类型
- 工具选择支持即时生效，无需确认按钮（已恢复原始版本）
- 分类筛选支持父子层级关系，提供更精确的工具定位
- 所有错误都会在控制台输出警告，便于开发调试
- 生产环境替换API即可，无需修改代码
