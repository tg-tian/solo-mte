# 支持搜索并应用流程模板数据

## 背景

模板库`https://lctemplates.gitlink.org.cn/`中记录了各种模板数据。现在，流程编排设计器需要集成模板功能。

## 交互流程

需要在流程编排设计器的“保存”按钮的左侧新增一个搜索框。

1. 用户在搜索框中输入搜索关键字，回车；
2. 系统从模板库中查询相关的模板，在搜索框下方以一个浮动下拉框的形式，将查询到的模板以列表的形式展示出来；
3. 用户点击一个模板条目；
4. 系统弹框询问用户是否应用；
5. 用户点击确定；
6. 系统将模板数据中的节点和边替换到当前设计器中。

## 模板库接口文档

``` yaml
---
openapi: 3.0.1
info:
  title: Template API
  version: "1.0"
  description: Template management API documentation
servers:
  - url: "/"
    description: Current server
paths:
  "/templates":
    get:
      summary: 获取模板列表
      description: 返回分页的模板列表，支持关键词搜索和kind分类过滤
      tags:
        - Templates
      parameters:
        - name: page
          in: query
          schema:
            type: integer
          description: 页码
        - name: per
          in: query
          schema:
            type: integer
          description: 每页数量
        - name: q[tag_fuzzy]
          in: query
          schema:
            type: string
          description: 搜索关键词（支持template_index和标签名称模糊匹配）
        - name: schema
          in: query
          schema:
            type: string
          description: 按schema分类过滤（如 inBuilder）
        - name: domain
          in: query
          schema:
            type: string
          description: 按domain分类过滤（如 通用）
        - name: template_type
          in: query
          schema:
            type: string
          description: 按template_type分类过滤
        - name: language_framework
          in: query
          schema:
            type: string
          description: 按language_framework分类过滤
        - name: file_extension
          in: query
          schema:
            type: string
          description: 按file_extension分类过滤
        - name: function
          in: query
          schema:
            type: string
          description: 按function分类过滤
      responses:
        "200":
          description: 成功
          content:
            application/json:
              schema:
                type: object
                properties:
                  templates:
                    type: array
                    items:
                      "$ref": "#/components/schemas/Template"
components:
  schemas:
    Template:
      type: object
      properties:
        id:
          type: integer
        name:
          type: string
        template_index:
          type: string
        template_description:
          type: string
        example_image_url:
          type: string
        code_url:
          type: string
        repository_url:
          type: string
        file_source:
          type: string
        submitter:
          type: string
        license:
          type: string
        created_at:
          type: string
          format: date-time
        updated_at:
          type: string
          format: date-time
        tags:
          type: object
          description: 按kind分组的标签
          example:
            schema:
              - inBuilder
            domain:
              - 通用
    TemplateInput:
      type: object
      required:
        - name
      properties:
        template_index:
          type: string
        template_description:
          type: string
        example_image_url:
          type: string
        code_url:
          type: string
        repository_url:
          type: string
        file_source:
          type: string
        submitter:
          type: string
        update_time:
          type: string
          format: date-time
        license:
          type: string
        name:
          type: string
```

你只需要关注“获取模板列表”接口即可，示例如下：

``` bash
curl -X 'GET' \
  'https://lctemplates.gitlink.org.cn/templates?page=0&per=999&q%5Btag_fuzzy%5D=%E8%AE%BE%E5%A4%87%E4%BA%8B%E4%BB%B6&schema=inBuilderFlow&template_type=eventflow' \
  -H 'accept: application/json'
```

**重点**：`schema`参数固定值为`inBuilderFlow`，`template_type`固定值为`eventflow`，通过以上两个参数限定只搜索流程模板，目前先不分页所以`page=0`且`per=999`，唯一变化的就是`q[tag_fuzzy]`参数，对应搜索框中的用户输入。

## 新页面元素的显示效果

1. 下拉列表功能优先考虑使用`packages\flow-devkit\lib\third-party\tdesign-exports.ts`中引入的`tdesign-vue-next`腾讯开源组件库的组件实现；
2. 每一个模板条目需要显示：模板名称、模板编号、模板描述，其中
   1. 模板名称更语义化，作为显示的重点
   2. 模板编号考虑适当淡化
   3. 模板描述自己独立一行显示
   4. 列表按照模板编号从小到大排序
3. 询问用户是否应用的弹框优先使用FarrisVue中的组件实现
4. 搜索框中含有文本时，右侧显示一个“clear”图标，点击则清空
