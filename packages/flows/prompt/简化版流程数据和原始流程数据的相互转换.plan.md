# 简化版流程数据和原始流程数据的相互转换

## 背景

本项目的核心是一个流程编排设计器，类似于Dify、Coze中的流程设计。

目前可以在设计器`packages\flow-designer`中通过可视化拖拉拽的方式编排流程，但是用户提出新需求：通过对话框和大模型对话，用自然语言描述流程，由大模型直接生成流程数据。

由于原始的流程数据较为复杂，其中包含了很多不必要的数据，循环节点与子节点还是直接嵌套的关系，对大模型不友好。为了减轻大模型的负担并提高准确性，我们设计了一套简化版的数据格式，大模型直接生成简化版的数据即可。

但是，流程设计器需要原始版本的流程数据才能运行，所以新增一套简化数据和原始数据的双向转换程序。

## 重点上下文指引

简化版流程数据的生成指南：`prompt\generate-flow-data\generate-flow-data.md`

原始流程数据的类型：`packages\flow-devkit\lib\types\flow-metadata.ts`中的`FlowMetadata`，转换时仅处理`nodes`和`edges`即可，其它字段无需转换

内置节点：`packages\flow-designer\src\components\builtin-nodes`，仅关注简化版文档中提到的节点即可

扩展一个流程节点需要提供的内容：`packages\flow-devkit\lib\types\node-definition.ts`中的`NodeDefinition`，所有的内置节点也都符合该类型定义

各种表达式：`packages\flow-devkit\lib\types\value-express.ts`

参数类型、类型详情的ts类型：`packages\flow-devkit\lib\types\type.ts`；基础的`TypeRefer`：`packages\flow-devkit\lib\types\basic-type-refer.ts`

在设计器中拖拽添加节点的实现：`packages\flow-designer\src\hooks\use-drag-new-node.ts`

关于设备调用和设备事件监听节点，设备元数据是通过`packages\flow-devkit\lib\hooks\use-device-info.ts`动态获取的；设备元数据的类型声明：`packages\flow-devkit\lib\types\device.ts`

关于函数调用节点或函数调用表达式所需的函数类型详情，是通过`packages\flow-devkit\lib\hooks\use-method-types.ts`获取的，提供给AI的函数类型是据此经过简化的

由于本项目代码中没有具体节点的类型声明，所以我临时补充了一个`prompt\all-nodes.ts.txt`仅供参考

另外，`prompt\prompts-to-generate-simple-flow-data-format`中的文件是用来生成`generate-flow-data.md`的提示词，你也可以参考一下

### 补充说明

1. 不需要关注`NodeGraphMeta`中的deletable、selectable、draggable字段
2. 不需要关注`FlowEdge`中的graphMeta字段
3. 不需要关注`Parameter`中的name、description、value、required、schema、inputHelp字段

## 目标

在`packages\flow-designer`中，实现简化版流程数据和原始流程数据的双向转换

## 后续目标

在`packages\flow-designer\src\hooks\use-ai-chat-panel.tsx`中集成AI对话的逻辑，当用户通过对话框用自然语言描述流程后，通过mock模拟大模型返回了生成的简化版流程数据，转化为原始流程数据并将之显示到设计器中
